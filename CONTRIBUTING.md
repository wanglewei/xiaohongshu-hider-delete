# 贡献指南

感谢您对小红书笔记批量管理工具的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题
- 使用GitHub Issues报告bug
- 提供详细的问题描述和重现步骤
- 包含系统环境信息（操作系统、Python版本等）
- 附上相关的错误日志

### 功能建议
- 在Issues中描述新功能的想法和使用场景
- 说明功能的实现价值和预期效果
- 欢迎提供设计思路或实现方案

### 代码贡献
1. Fork本项目到您的GitHub账户
2. 创建功能分支：`git checkout -b feature/新功能名称`
3. 提交更改：`git commit -am '添加新功能'`
4. 推送分支：`git push origin feature/新功能名称`
5. 创建Pull Request

## 开发环境设置

### 环境要求
- Python 3.7+
- Chrome浏览器
- Git

### 安装步骤
```bash
# 克隆项目
git clone https://github.com/您的用户名/xiaohongshu-note-manager.git
cd xiaohongshu-note-manager

# 安装依赖
pip install -r requirements.txt

# 运行测试
python -m pytest tests/
```

## 代码规范

### Python代码风格
- 遵循PEP 8规范
- 使用4个空格缩进
- 行长度不超过88字符
- 函数和类添加类型注解

### 注释规范
- 所有公共函数必须有docstring
- 复杂逻辑添加行内注释
- 注释使用中文，与项目保持一致

### 提交信息规范
```
类型(范围): 简短描述

详细描述（可选）

相关问题: #issue编号
```

类型包括：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

## 测试

### 运行测试
```bash
# 运行所有测试
python -m pytest

# 运行特定测试文件
python -m pytest tests/test_scraper.py

# 生成覆盖率报告
python -m pytest --cov=.
```

### 编写测试
- 为新功能编写单元测试
- 测试文件命名：`test_*.py`
- 测试函数命名：`test_*`
- 使用描述性的测试名称

## 文档

### 更新文档
- 新功能需要更新相关文档
- API变更需要更新README
- 重要变更需要更新CHANGELOG

### 文档风格
- 使用清晰简洁的中文
- 提供代码示例
- 包含必要的截图或图表

## 发布流程

### 版本号规范
遵循语义化版本控制（SemVer）：
- `MAJOR.MINOR.PATCH`
- 主版本号：不兼容的API修改
- 次版本号：向下兼容的功能性新增
- 修订号：向下兼容的问题修正

### 发布检查清单
- [ ] 所有测试通过
- [ ] 文档已更新
- [ ] CHANGELOG已更新
- [ ] 版本号已更新
- [ ] 创建Git标签
